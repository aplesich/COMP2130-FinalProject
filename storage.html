localStorage



Definition

localStorage is a method we may use to store data locally within the user's browser with no expiration date. localStroage will hold the data, with no expiry date, as a result the data will not be deleted even if the browser is closed and opened again.
localStorage can be accessed rather quickly compared to back end storage methods, that is why it is an ideal way for a user to store a small amount of relevant data they want to access later.

Benefits

  • It stores up to 5-10MB of data , the amount of storage allowed is dependant on the browser
  • Easy to use – no need for a web server or backend database.
  • Great for getting an idea up and running and for testing purposes.

Cautions

  • Can only be text, data will always be a “read-only” format
  • Anything stored with localstorage is not secure, because it can be accessed relatively quickly with code. As a result, we should never store private or personal information through localStorage.

Syntax

Add data

To add the data in localStorage, both key and value are required to pass in setItem() function.

  • localStorage.setItem("girl", "Helen");

Retrieve data

A key is required to retrieve the data from storage and a JavaScript variable to store the returned data.

  • var storedData = localStorage.getItem("girl");

Remove data

The same key is required to remove the value attached with it.

  • localStorage.removeItem("girl");

Clear localStorage

Using a simple clear() function, it will remove all the localStorage data:

  • localStorage.clear()
