localStorage



Definition

localStorage is a method we may use to store data locally within the user's browser with no expiration date. localStroage will hold the data, with no expiry date, as a result the data will not be deleted even if the browser is closed and opened again.
localStorage can be accessed rather quickly compared to any back end storage methods, that is why it is an ideal way for a user to store a small amount of relevant data they want to access later.

Benefits

  • Storage capacity of up to 5-10MB of data ,note that the amount of storage allowed is dependant on the browser
  • localStorage is easy to use, and there is no need to involve a backend database
  • localStorage is perfect for getting things stored for testing purposes.

Cautions

  • Can only be text, data will always be a “read-only” format
  • Anything stored with localStorage is not secure, because it can be accessed relatively quickly using code. As a result, we should never store private or personal information through localStorage.

Syntax

Adding new data to localStorage

To add the data in localStorage, both a key and value are required to pass the setItem() function.

  • localStorage.setItem("girl", "Helen");

Retrieving stored data

Using a key is required to retrieve your stored data, you will use a JavaScript variable to store the returned data.

  • var storedData = localStorage.getItem("girl");

Removing stored data

The same key is required to remove the value attached with it.

  • localStorage.removeItem("girl");

Clearing localStorage

Using a simple clear() function, it will remove all the localStorage data:

  • localStorage.clear()
